# ADR 002: String Template Approach for WebView HTML Generation

## Status
Accepted

## Context
VS Code extensions need to generate HTML for WebView panels. Traditional approaches use static HTML files or templating engines.

## Decision
Generate all WebView HTML, CSS, and JavaScript as string templates in TypeScript files.

## Rationale
1. **No bundler complexity**: No webpack/vite configuration needed
2. **Type safety**: All content is generated by TypeScript code
3. **Dynamic theming**: Easy to inject VS Code theme colors at runtime
4. **CSP compliance**: Nonces and CSP headers are generated programmatically

## Consequences
Positive:
- Simple build process
- Full TypeScript type checking
- Easy to customize per-extension

Negative:
- Larger TypeScript files
- No syntax highlighting for HTML/CSS/JS inside strings
- Harder to use standard frontend tooling
